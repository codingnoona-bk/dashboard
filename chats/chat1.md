## User

선주 지적(owner punch-item) 관리 시스템의 UI를 디자인해줘.
조선소 현장에서 들어오는 선주 지적을 한 곳에서 등록·배정·추적해 "누락 없이" 끝까지 관리하는 웹 대시보드야.

## 핵심 문제
지적이 엑셀·카카오톡 등 여러 채널로 산발적으로 들어와서, 지금 전체가 몇 건이고
누가 처리 중이고 끝났는지 추적이 안 되고 누락이 생긴다. 이걸 단일 창구로 해결한다.

## 사용자 & 기기 (★ 화면을 기기에 맞게 다르게 설계)
1. 현장 작업자 — 주로 "태블릿" 사용. 배정받은 지적 확인 → 착수 → 완료 보고.
   → 터치 우선. 큰 탭 영역, 큰 버튼, 단순한 1열 흐름, 한 손/장갑 낀 손으로도 누르기 쉽게.
     불필요한 정보 제거하고 "내 업무"와 상태 변경 버튼을 화면 중심에 크게.
2. 업무 담당자 — 데스크탑. 지적 등록·확인·배정·전체 현황 모니터링.
   → 정보 밀도 높은 대시보드 + 목록/필터.
3. 승인자 — 주로 "데스크탑" 사용. 완료 보고된 건을 검토 후 승인/반려.
   → 정보 밀도 높게. 좌측 목록 + 우측 상세(작업 사진·메모) 2단 분할, 일괄 검토에 유리하게.

## 상태 흐름 (배지·색으로 한눈에 구분)
등록됨 → 확인됨 → 배정됨 → 진행중 → 검토대기 → 완료(종결)
(검토대기에서 승인자가 "반려" 시 → 진행중으로 회귀. 별도 "지연" 강조 상태)
- 검토대기 → 완료 전이는 반드시 승인자의 명시적 승인을 거친다(작업자가 혼자 닫을 수 없음).
- 상태 색: 등록됨=중립 그레이, 확인됨=블루, 진행중=브랜드 퍼플, 검토대기=앰버,
  완료=그린, 반려/지연=레드.

## 만들 화면 (우선순위 순)
1. 대시보드(담당자/데스크탑): 상단 요약 카드(전체·진행중·검토대기·완료·지연),
   상태별 칸반 또는 막대, 지연 건 강조 영역, 담당자/호선/구역/상태 빠른 필터.
2. 내 업무 화면(작업자/태블릿): 나에게 배정된 지적만, 상태별 정렬,
   큰 버튼으로 "착수 / 완료 보고", 작업 사진·메모 첨부.
3. 승인 화면(승인자/데스크탑): 검토대기 건만 모아 좌측 목록 + 우측 상세,
   작업 결과(사진·메모) 확인 후 승인 / 반려(사유 입력 필수).
4. 지적 등록 화면(담당자): 지적 내용·호선/구역·발생일·출처(엑셀/카톡 등)·우선순위·첨부,
   필수 항목 미입력 시 등록 불가.
5. 지적 상세/배정 화면: 전체 정보 + 첨부 + 상태 변경 이력 타임라인 + 작업자 배정/기한 설정 + 코멘트.

## 디자인 원칙 (코알누 브랜드, 첨부 이미지 참고)
- 시그니처 컬러: 밝은 퍼플/바이올렛(대략 #7C5CFC)을 포인트로. 배경은 깨끗한 화이트, 여백 넉넉하게.
- 버튼: 완전히 둥근 pill 형태, 퍼플 채움 + 흰 볼드 텍스트. 부드럽고 친근한 느낌.
- 타이포: 굵고 명확한 헤딩(Pretendard 계열), 한글 본문도 깔끔한 산세리프.
- 톤: 친근하고 모던. 차갑고 딱딱한 산업용 느낌이 아니라, 깨끗하고 따뜻한 도구.
- 단, 운영 도구인 만큼 "상태와 건수가 한눈에 보이는 스캔성"을 최우선으로. 장식보다 가독성.
- 로고/디테일: 코드 브래킷 모티프({ ; }) 느낌의 퍼플 액센트를 가볍게 활용 가능.

## 성공 기준 (디자인이 충족해야 할 것)
- 대시보드 한 화면에서 "지금 미완료 몇 건? 지연 몇 건? 누가 처리 중?"이 즉시 보일 것.
- 모든 지적이 현재 상태·담당자와 함께 식별될 것.
- 태블릿(작업자)에서는 터치만으로 상태 변경이 쉽고, 데스크탑(승인자)에서는 빠른 일괄 검토가 가능할 것.
